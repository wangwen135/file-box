# File Box 需求文档

> 参考 Simple-Pic 项目的设计与实现，增强 File Box 的用户管理和存储空间功能

**版本：** v0.2.0  
**日期：** 2026-03-21  
**状态：** 待评审

---

## 1. 项目概述

File Box 是一个文件共享和管理工具。当前版本已支持基础的文件上传、用户分组、匿名访问等功能。

本次需求旨在引入 **存储空间** 概念，借鉴 Simple-Pic 项目的多存储空间设计，实现更灵活的文件管理和权限控制。

---

## 2. 用户角色定义

| 角色 | 权限说明 |
|------|----------|
| **Admin（超级管理员）** | 拥有最高权限，可进入管理界面，管理存储空间、用户、查看所有文件。配置存储在配置文件中 |
| **管理员（Manager）** | 可管理指定存储空间内的文件（上传、删除），不可管理存储空间和用户 |
| **普通用户（User）** | 仅可上传文件到被授权的存储空间，不可删除文件 |
| **匿名用户** | 无需登录，可访问匿名存储空间，默认**不允许上传** |

---

## 3. 核心需求

### 3.1 存储空间管理

#### 3.1.1 存储空间概念

- 每个存储空间是一个独立的文件目录
- 每个存储空间可配置：
  - 名称（唯一标识）
  - 物理路径
  - 最大容量限制
  - 是否允许匿名访问

#### 3.1.2 Admin 权限

Admin 用户可在**管理界面**执行以下操作：

| 操作 | 说明 |
|------|------|
| 新增存储空间 | 创建新的存储目录和配置 |
| 编辑存储空间 | 修改名称、容量等配置 |
| 删除存储空间 | 删除配置（可选是否删除物理文件） |
| 查看存储空间统计 | 已用容量、文件数量等 |

**注意：** 所有存储空间配置存储在 `application.yml` 配置文件中，无需数据库。

#### 3.1.3 匿名存储空间

- 匿名存储空间是一个特殊的存储空间
- 配置项：`anonymous.enabled` 控制是否启用
- 匿名用户可直接访问，无需登录
- 可配置匿名用户是否允许上传

---

### 3.2 用户管理

#### 3.2.1 Admin 权限

Admin 用户可在**管理界面**执行以下操作：

| 操作 | 说明 |
|------|------|
| 新增用户 | 创建用户账号，设置用户名、密码、角色 |
| 编辑用户 | 修改用户信息、角色、关联存储空间 |
| 删除用户 | 删除用户账号 |
| 为用户指定存储空间 | 关联用户与一个或多个存储空间 |

**注意：** 所有用户配置存储在 `application.yml` 配置文件中，无需数据库。

#### 3.2.2 用户 - 存储空间关联

- 一个用户可以关联多个存储空间
- 用户登录后可切换当前使用的存储空间
- 用户只能访问被授权的存储空间

---

### 3.3 文件操作权限

| 角色 | 上传 | 删除 | 查看 | 管理存储空间 | 管理用户 |
|------|------|------|------|--------------|----------|
| **Admin** | ✅ | ✅ | ✅ | ✅ | ✅ |
| **管理员** | ✅ | ✅ (仅当前空间) | ✅ (仅当前空间) | ❌ | ❌ |
| **普通用户** | ✅ | ❌ | ✅ (仅当前空间) | ❌ | ❌ |
| **匿名用户** | ❌ (默认关闭) | ❌ | ✅ (仅匿名空间) | ❌ | ❌ |

---

### 3.3.1 存储限额配置

- 存储限额直接配置在存储空间配置中
- 每个存储空间独立配置 `max-size`
- 达到限额后禁止上传并提示

---

### 3.4 配置文件结构

**配置生成流程：**

1. 默认打包时**不包含**配置文件
2. 首次启动后，使用 Admin 账号登录
3. 登录成功后自动生成 `application.yml` 配置文件
4. 后续启动从配置文件加载用户和存储空间配置

**参考 Simple-Pic 的 YAML 配置结构：**

```yaml
file-box:
  # 系统基础信息
  system:
    name: File Box
    description: 文件共享和管理工具

  # 匿名访问配置
  anonymous:
    enabled: true
    name: 匿名用户组
    storage-space: anonymous  # 关联的存储空间
    allow-upload: false       # 是否允许匿名用户上传（默认关闭）

  # 存储空间配置
  storage-spaces:
    - name: default
      path: /opt/file-box/default
      max-size: 10GB    - name: shared
      path: /opt/file-box/shared
      max-size: 50GB    - name: anonymous
      path: /opt/file-box/anonymous
      max-size: 5GB
  # 用户配置
  users:
    - username: admin
      password: {BCrypt 加密密码}
      role: ADMIN
      storage-spaces: [default, shared, anonymous]  # 可访问所有空间
    - username: manager1
      password: {BCrypt 加密密码}
      role: MANAGER
      storage-spaces: [shared]  # 只能管理 shared 空间
    - username: user1
      password: {BCrypt 加密密码}
      role: USER
      storage-spaces: [default]  # 只能访问 default 空间
```

## 4. 功能模块设计

### 4.1 认证与权限模块

**现有功能：**
- ✅ 登录页面
- ✅ 用户分组配置
- ✅ 基础角色（user/admin）

**需要增强：**
- ⏳ 引入 BCrypt 密码加密（参考 Simple-Pic）
- ⏳ 会话管理优化（ConcurrentHashMap 存储，30 天有效期）
- ⏳ 权限拦截器细化到存储空间级别

### 4.2 存储空间模块

**新增功能：**
- ⏳ 存储空间配置管理（Admin 专属）
- ⏳ 存储空间切换（用户登录后）
- ⏳ 存储空间使用统计
- ⏳ 容量超限检查和提示

**技术实现参考 Simple-Pic：**
- 使用 File API 获取文件大小和修改时间
- 缓存存储统计信息（5 分钟 TTL）
- 支持的单位：KB, MB, GB

### 4.3 用户管理模块

**新增功能：**
- ⏳ 用户管理后台页面（Admin 专属）
- ⏳ 创建/编辑/删除用户
- ⏳ 为用户分配存储空间权限
- ⏳ 用户角色管理（ADMIN/MANAGER/USER）

### 4.4 文件上传模块

**现有功能：**
- ✅ 多种上传方式（粘贴、拖拽、选择）
- ✅ 上传进度显示
- ✅ 文本上传

**需要增强：**
- ⏳ 上传时验证当前存储空间的容量限制
- ⏳ 上传时验证用户权限
- ⏳ 存储路径优化：`{存储空间}/{年}/{月}/{UUID}.{后缀}`

### 4.5 文件管理模块

**现有功能：**
- ✅ 文件列表展示
- ✅ 按年月筛选

**需要增强：**
- ⏳ 管理员可删除文件（当前存储空间内）
- ⏳ Admin 可删除任意文件
- ⏳ 文件操作权限校验

### 4.6 后台管理模块（新增）

**Admin 专属管理界面：**

| 页面 | 功能 |
|------|------|
| 仪表盘 | 系统概览、存储统计、用户统计 |
| 存储空间管理 | 增删改查存储空间 |
| 用户管理 | 增删改查用户，分配存储空间 |
| 系统设置 | 基础配置、安全配置 |

**注意：** 所有配置修改后直接写入 `application.yml` 文件，无需数据库。

## 5. 技术实现要点

### 5.1 参考 Simple-Pic 的设计

| 模块 | Simple-Pic 实现 | File Box 可借鉴 |
|------|----------------|-----------------|
| 配置管理 | 首次启动生成配置文件 + SnakeYAML | ✅ 采用相同方案 |
| 密码加密 | BCrypt | ⏳ 需要引入 |
| 会话管理 | ConcurrentHashMap + 定时清理 | ⏳ 可参考实现 |
| 存储统计 | File API + 缓存 | ⏳ 可参考实现 |
| 权限控制 | 角色 + 拦截器 | ⏳ 需要细化到存储空间 |

### 5.2 需要新增的依赖

```xml
<!-- BCrypt 密码加密 -->
<dependency>
    <groupId>org.springframework.security</groupId>
    <artifactId>spring-security-crypto</artifactId>
</dependency>

<!-- Guava（可选，用于限流和缓存） -->
<dependency>
    <groupId>com.google.guava</groupId>
    <artifactId>guava</artifactId>
    <version>31.1-jre</version>
</dependency>
```

### 5.3 数据模型设计

**新增实体类：**

```java
// 存储空间
public class StorageSpace {
    private String name;
    private String path;
    private Long maxSize;      // 字节
    private String domain;
}

// 用户角色枚举
public enum Role {
    ADMIN,      // 超级管理员
    MANAGER,    // 存储空间管理员
    USER        // 普通用户
}

// 用户
public class User {
    private String username;
    private String password;   // BCrypt 加密
    private Role role;
    private List<String> storageSpaces;  // 可访问的存储空间
}
```

## 7. 风险与注意事项

### 7.1 数据迁移

- 现有用户配置需要迁移到新的结构
- 现有文件目录需要重新组织到存储空间下
- 建议提供迁移脚本或向导

### 7.2 向后兼容

- 保留现有 `groups` 配置的兼容性
- 提供配置迁移指南

### 7.3 安全性

- Admin 接口必须严格权限验证
- 存储空间路径需要防止目录穿越攻击
- 文件上传需要验证文件类型和大小

## 8. 验收标准

### 8.1 功能验收

- [ ] Admin 可创建、编辑、删除存储空间
- [ ] Admin 可创建、编辑、删除用户
- [ ] Admin 可为用户分配存储空间权限
- [ ] 普通用户只能上传文件到授权空间
- [ ] 管理员可删除当前空间内的文件
- [ ] 匿名用户可访问匿名空间
- [ ] 存储空间容量限制生效

### 8.2 性能验收

- [ ] 存储空间统计响应时间 < 500ms
- [ ] 文件列表加载时间 < 1s（1000 文件内）
- [ ] 并发上传稳定（10 并发）

### 8.3 安全验收

- [ ] 未授权访问被拦截
- [ ] 密码加密存储
- [ ] 会话安全过期
- [ ] 防止目录穿越攻击

## 10. 已确认事项

1. **角色定义：** 三个角色 - Admin（超级管理员）、Manager（普通管理员）、User（一般用户），另加匿名用户
2. **存储配额：** 硬限制，配置在存储空间管理中，超过禁止上传
3. **匿名用户上传：** 默认关闭
4. **数据迁移：** 无需迁移，直接配置一个存储空间指向现有目录即可
5. **配置存储：** 所有配置存储在 `application.yml` 文件中，无需数据库和中间件
6. **管理界面：** Admin 可进入专属管理界面进行配置管理
